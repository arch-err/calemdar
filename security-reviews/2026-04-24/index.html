    calemdar security review¶

    Commit: master @ /home/archerr/Git/arch-err/Tools/calemdar Threat model: single-user Linux daemon on its own files. No multi-tenant concerns.

    Executive summary¶

    Overall posture is good for the stated threat model. Path handling is defensive: slugs are sanitised to [a-z0-9-], calendars are validated against an allowlist before any EventPath call, dates must parse as YYYY-MM-DD before being substituted into filenames, and base_path rejects ... YAML parsing is yaml.v3 (has some built-in safeguards; no explicit depth/size caps beyond a 16MiB scanner buffer). The single most important finding is a credential-in-log leak in internal/notify: the ntfy URL is logged verbatim on every push, and the project's own documentation tells users to bake auth into the URL. No critical or high findings.

    Findings¶

    Critical¶

    None.

    High¶

    None.

    Medium¶

    M1 — ntfy URL leaks basic-auth credentials to logs/stdout. internal/notify/notify.go:101, cmd/calemdar/cmd_notify.go:41. n.cfg.NtfyURL is written verbatim to log.Printf on every successful push (… → %s/%s"), and to stdout in notify test. The in-code comment at internal/notify/notify.go:107–108 explicitly instructs users to "bake auth into the URL" for private topics, so a https://user:pass@ntfy.example.com URL is a realistic config. Those creds then land in systemd-journal / tty scrollback. Fix: scrub userinfo before logging — parse with net/url and log u.Redacted() (stdlib, elides password). Applies to both call sites.

    Low¶

    L1 — base_path validation allows absolute paths. internal/config/config.go:99–101. Validate only rejects ... An absolute base_path like /etc works in practice because filepath.Join(Root, "/etc") → Root/etc (Join does not honor absolute-path semantics on its non-first argument). Not exploitable today, but footgun-shaped — a future refactor that uses path.Clean+filepath.Rel or a different joiner could turn this into a traversal. Cheap to harden now. Fix: reject base_path starting with / (or any OS separator) and reject strings.HasPrefix(base_path, string(os.PathSeparator)).

    L2 — NtfyTopic injected into URL path without validation or escaping. internal/notify/notify.go:119,143, internal/config/config.go:124–126. Topic is concatenated raw: TrimRight(NtfyURL,"/") + "/" + NtfyTopic. Config only requires non-empty/non-whitespace. Topic containing ../admin, ?query=1, or #frag would rewrite the POST target. Threat model is single-user so this is self-inflicted, but a shape check (^[A-Za-z0-9_-]{1,64}$ per ntfy's own allowed charset) is a one-liner that eliminates the surprise. Fix: validate topic regex in Notifications.Validate.

    Informational¶

    I1 — YAML unmarshalling has no explicit resource caps. internal/model/parse.go:73,97, internal/fcparse/translate.go:27,44,61, internal/config/config.go:152. yaml.v3 is called on user-controlled file contents with no depth/size wrapper. The underlying library does cap alias expansion (mitigating classic billion-laughs), and the frontmatter is bounded by a 16MiB scanner buffer (parse.go:22, translate.go:225), so the practical blast radius is bounded memory on parse. Noted for the next reviewer; no action recommended under current threat model.

    I2 — fsnotify self-write suppression can be spoofed by matching mtime. internal/watch/watch.go:290–311. isRecentSelfWrite suppresses if info.ModTime().Equal(m.mtime). An attacker with write access to the vault can call utimensat to match — file changes would be silently ignored by the daemon. Out of scope (requires local write access), but the mechanism is exactly the one the comment at watch.go:78–82 worries about, so documenting: yes, spoofable.

    I3 — $EDITOR split by whitespace for config edit. cmd/calemdar/cmd_config.go:123–128. strings.Fields($EDITOR) then exec.Command(parts[0], parts[1:]...). Attacker controls their own env; noted as acknowledged per the brief. Not command-injection in the classic sense (no shell invoked), but values like EDITOR="vim --cmd ':!rm -rf ~'" would pass the option through to the editor. Same single-user caveat.

    I4 — .calemdar/cache.db created with driver defaults (umask-dependent). internal/store/store.go:30–38. No explicit Chmod. On a default umask 022 this yields 0644. Fine for single-user, noted.

    I5 — Unused-entry GC in selfWrites map. internal/watch/watch.go:63–64,135–146. Marks are deleted on TTL expiry only when something queries them. Marks for files that are never re-touched accumulate. Not a race; not a leak of security concern. Memory-bounded by workload. No action.

    Surfaces reviewed clean¶

    • vault.Resolve, vault.EventPath, vault.Scaffold — all path construction rooted in validated or on-disk inputs; slug/calendar/date are either allowlisted or regex-constrained before joining.
    • fcparse.Slugify (internal/fcparse/slug.go) — output is strictly [a-z0-9-], no traversal payload possible.
    • reconcile.Series — writes through v.EventPath(r.Calendar, e.Date, r.Slug). r.Calendar is validated by expand.Expand (internal/expand/expand.go:24) before any path join; e.Date is parsed; r.Slug for disk-loaded roots is filepath.Base(path) which cannot contain separators.
    • reactor.migrateOne — calendar is derived from the on-disk relative path under events/, then re-validated against model.ValidCalendar; slug is fcparse.Slugify output; target existence is checked before write.
    • archive.Run — year is e.Date[:4] after ParseDate success; calendar is calendarFromPath derived from on-disk path under EventsDir.
    • cmd_event new / cmd_series new — slug from Slugify, calendar from askChoice(model.Calendars), date from ParseDate. No user string reaches a path unsanitised.
    • store/store.go — all SQL uses ? placeholders; ListUpcoming builds IN (...) with generated placeholders and args slice, no concat of user strings. All Query sites have matching defer rows.Close(). Connection is *sql.DB (pooled), Close() shuts it down.
    • store.Reindex — walks disk with filepath.WalkDir, calendar from on-disk path.
    • notify HTTP client — uses default Go http.Transport (TLS verification enabled), no custom TLSClientConfig, response body drained and closed on every path.
    • watch.Watcher — selfMu, pendingMu consistently guard their maps; the documented debounce-fire-time recheck at watch.go:249–251 closes the NotifySelfWrite-vs-raw-event race. flushPending runs under lock.
    • writer.WriteEvent / WriteRoot — path is caller-controlled but every caller routes through vetted path construction (see above). Mkdir + atomic-ish os.WriteFile.
